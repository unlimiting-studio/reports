1. 시작점 — 무엇을 만들고자 했나

Qyuni의 1차 목표는 수익 극대화가 아니라 검증 가능한 거래 시스템이에요. 350만 원은 본격 운용 자본이 아니라 가설 검증·인프라 신뢰도 확보를 위한 학습 시드로 두고, 단계별로 백테스트 → 페이퍼 → 소액 실전 순서로 자본 노출을 늘려가는 그림.

  • • 전략 가족: 차트형 그리드 단일을 첫 칼로. 분~시간 단위 보유, OHLCV로 시뮬 가능, 검증 사이클이 짧음.
  • • universe: 고정 N을 박지 않고, 바이낸스 24h quoteVolume 상위에서 동적으로 추출 + 친화도 임계로 거름.
  • • 자본 운용: 한 심볼에 350만 다 박지 않고, 통과 N에 균등 분할.
  • • 재현성·운영 안전: 모든 산출물에 git hash + dirty flag + 데이터 sha256 + 거래소 응답 시각. 폴백/바이패스 금지, fail-fast 원칙.
  • • API 키 정책: 출금 권한 영구 OFF + IP 화이트리스트 + 매매·잔고만. 평문이어도 키 자체로 자산을 빼갈 수 없게 잠금.

2. 합의 — 핑퐁으로 잠근 의사결정

항목 결정 근거
목표 검증 가능한 거래 시스템 수익 극대화 ❌. 350만 원은 학습·검증 시드.
첫 전략 차트형 그리드 분~시간 단위 보유, OHLCV로 시뮬 가능.
펀딩비 차익 첫 칼에서 제외 며칠 보유 = "짧은 거래" 결과 안 맞음.
스프레드 백테스트 ❌, 페이퍼 ⭕ OHLCV로는 자기실현·레그 리스크 못 잡음.
Universe 고정 N ❌, 동적 셀렉션 ⭕ 시장 상태에 따라 통과 N이 자연 조절.
자본 운용 선정 N개 균등 분할 한 심볼에 350만 다 박지 않음.

4. 시스템 그림

1. Universe        ─→  fetch_top_volume_pairs_with_metadata()  (or snapshot)
2. Data            ─→  load_or_fetch()  +  DataProvenance(sha256, fetch_ts)
3. Screening       ─→  score_grid_friendliness()  ─→  passes_thresholds()
4. Selection       ─→  N개 통과 (시장 상태 의존)
5. Capital         ─→  CAPITAL_KRW / N  균등 분할
6. Backtest        ─→  GridConfig + BacktestParams(fee, slip, vol_part, min_notional)
                       ↳ run_grid_backtest()  ─→  RoundTrip 추적
7. Walk-forward    ─→  4 fold rolling (train 40% / test 15%)
8. Report          ─→  run_metadata + git_dirty + walk_forward_summary
                       data_provenance + universe_snapshot_path

현재 구조의 한계는 백테스트 전용이라는 점. 페이퍼/실전을 얹으려면 어댑터 레이어 분리가 필요.

6. 솔직한 한계

  • +2.28%는 30일 한 구간 결과. 다른 시장 상태에서는 0이나 음수도 충분히 가능.
  • OHLCV 시뮬은 호가창·체결 순서·latency를 모름. 실전 결과 ≠ 백테스트 결과.
  • 0.1% 이상 갭을 한 거래에서 잡는 건 현실에서 어려움. 양쪽 수수료 합 0.25%(테이커-테이커)가 본전. 일반 시장에선 거의 안 열림.
  • 현재 코드는 백테스트 전용. 페이퍼/실전을 얹으려면 어댑터 분리 필요.
  • Risk Layer가 없음. 일일 손실 한도·drawdown 가드·kill switch 모두 미구현. 자산 들어가기 전 필수.

7. 앞으로의 방향

  1. Phase A · 1~2주

    어댑터 추상화

    `DataSource` / `ExchangeAdapter` / `Strategy` / `Clock` 인터페이스. 백테/페이퍼/실전이 같은 코어 + 다른 어댑터.

  2. Phase B · 1주

    Risk + Persistence

    `Risk.check()` 코어 게이트, SQLite로 trades/positions/orders 영속, 재시작 시 거래소 ↔ DB reconcile.

  3. Phase C · 1~2주

    페이퍼 트레이딩

    `LiveSource(WebSocket)` + `SimulatedExchange`. 라이브 데이터로 가상 매매. 호가창 캡처는 한국 거래소 tick 데이터 자산도 같이 누적.

  4. Phase D · 1주

    Observability + 운영

    structured log, Slack 알람, 실시간 대시보드, systemd graceful shutdown.

  5. Phase E · 2~4주

    소액 실전

    10~50만 원으로 시작. API 키는 매매 권한만 + IP 화이트리스트 + 출금 영구 OFF.

8. 운영 원칙 (잠금)

  • 🔐 API 키 권한 최소화: 출금 OFF, IP 화이트리스트, 매매·잔고만.
  • 🚫 폴백/바이패스 금지: fail-fast가 기본. 잘못된 입력은 즉시 ValueError.
  • 🧪 테스트 없는 변경 금지: 코어/risk/execution은 단위 테스트로 잠금.
  • 📜 모든 산출물에 provenance: git_commit + dirty flag + 데이터 hash + fetch 시각.
  • 🐢 실전 진입은 점진적: 백테 → 페이퍼(1주+ 무결) → 소액(10만) → 점진 확대.
